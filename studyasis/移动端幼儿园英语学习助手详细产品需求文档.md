# 移动端幼儿园英语学习助手详细产品需求文档

（适配TRAE编程工具开发使用）

## 1. 产品概述

### 1.1 产品定位

专为3-6岁幼儿园小朋友设计的轻量化英语学习辅助工具，通过"听音选图"的趣味互动形式，系统性锻炼幼儿英语听力和单词理解能力，构建"老师配置-学生练习-数据反馈"的完整教学闭环。

### 1.2 适用场景

* 幼儿园日常英语教学配套作业
* 家庭亲子英语互动练习
* 幼儿园英语教学效果评估辅助工具

### 1.3 技术实现要求

* 技术栈：优先使用零代码/低代码工具搭建，前端采用响应式设计
* 部署方式：云端部署，支持公网访问
* 数据存储：云端数据库，支持数据备份与恢复
* 成本控制：优先使用免费开源资源与工具

## 2. 用户角色与权限体系

| 用户角色  | 操作端       | 核心权限                     |
| ----- | --------- | ------------------------ |
| 幼儿园学生 | 移动端       | 接收作业、完成练习、查看个人学习记录       |
| 英语老师  | PC端       | 班级管理、题库管理、作业发布、学情查看、作业批改 |
| 系统管理员 | 后台管理端     | 账号管理、系统配置、数据统计、资源审核      |
| 家长    | 移动端（附属账号） | 协助孩子登录、查看孩子学习报告          |

## 5. 非功能性需求

### 5.1 性能需求

| 指标     | 要求                          |
| ------ | --------------------------- |
| 页面加载时间 | 移动端页面首屏加载≤2s，图片懒加载          |
| 操作响应时间 | 普通操作反馈≤500ms，提交类操作≤2s       |
| 发音播放   | 点击播放按钮后≤300ms开始播放           |
| 并发支持   | 支持单园所1000人同时在线使用，平台支持10万+用户 |
| 接口响应   | 95%的接口响应时间≤1s               |

### 5.2 兼容性需求

| 环境     | 支持版本                                       |
| ------ | ------------------------------------------ |
| 移动端系统  | iOS 12.0+，Android 8.0+                     |
| 移动端浏览器 | 微信内置浏览器、Safari、Chrome、UC浏览器等主流浏览器          |
| PC端浏览器 | Chrome 80+、Edge 80+、Firefox 75+、Safari 13+ |
| 屏幕适配   | 移动端支持320px-430px宽度屏幕，PC端支持1280px以上分辨率      |

### 5.3 可用性需求

| 指标     | 要求                            |
| ------ | ----------------------------- |
| 系统可用率  | ≥99.5%，年 downtime ≤43.8小时     |
| 故障恢复时间 | 一般故障≤30分钟，严重故障≤2小时            |
| 数据备份   | 每日自动备份，备份保留30天，支持一键恢复         |
| 操作容错   | 重要操作提供二次确认，支持撤销误操作            |
| 易用性    | 学生端操作步骤≤3步即可进入练习，教师端核心操作≤5步完成 |

### 5.4 安全性需求

| 类别   | 要求                              |
| ---- | ------------------------------- |
| 数据加密 | 用户敏感信息（手机号、密码）加密存储，传输过程使用HTTPS  |
| 数据隔离 | 不同园所、班级数据完全隔离，禁止跨权限访问           |
| 权限控制 | 三级权限体系，每个角色只能访问授权范围内的功能和数据      |
| 内容安全 | 所有上传的图片、音频资源经过审核，禁止违规内容         |
| 合规要求 | 符合《个人信息保护法》、《未成年人网络保护条例》等相关法规要求 |
| 防攻击  | 防止SQL注入、XSS攻击、CSRF攻击等常见网络攻击     |

### 5.5 体验需求

| 类别   | 要求                             |
| ---- | ------------------------------ |
| 视觉风格 | 学生端采用明亮柔和的马卡龙配色，卡通化设计，避免尖锐元素   |
| 交互设计 | 学生端操作简单，大按钮、大字体，减少文字输入，以点击操作为主 |
| 音效反馈 | 操作配有生动的音效反馈，正确/错误有不同音效提示       |
| 激励体系 | 采用勋章、等级、连续打卡奖励等激励机制，提升学习兴趣     |
| 无广告  | 学生端无任何商业广告，界面纯净                |

## 7. 接口设计规范（参考）

### 7.1 通用规范

* 接口采用RESTful风格设计
* 请求/响应数据格式为JSON
* 所有接口需要签名验证
* 响应统一格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

* 错误码规范：200-成功，400-参数错误，401-未登录，403-无权限，500-服务器错误

### 7.2 核心接口列表

| 接口地址                         | 请求方式 | 功能说明     |
| ---------------------------- | ---- | -------- |
| /api/user/login              | POST | 用户登录     |
| /api/user/info               | GET  | 获取用户信息   |
| /api/homework/today          | GET  | 获取今日作业   |
| /api/homework/detail         | GET  | 获取作业详情   |
| /api/homework/submit         | POST | 提交作业答案   |
| /api/homework/record         | GET  | 获取作业完成记录 |
| /api/word/list               | GET  | 获取单词列表   |
| /api/teacher/homework/create | POST | 老师创建作业   |
| /api/teacher/homework/list   | GET  | 老师获取作业列表 |
| /api/teacher/statistic       | GET  | 获取统计数据   |
